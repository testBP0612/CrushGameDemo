# Art Spec Sheet — 精確尺寸與製程（ART_CONTRACT v1.1 配套細則）

> 這是 `ART_CONTRACT.md`（v1.1 locked）的**配套細化**：把「建議尺寸」講到可直接照做，並列出製程清單。
> **不變更**合約鎖定項（透明/面向/檔名/路徑/匯入）；數值與合約一致，僅更精確。
> UI 尺寸表是**現況快照**（程式樣式），供對齊/未來皮膚用——**美術預設不需出按鈕圖**。

## 0. 畫布與安全區（1080×1920）
| 區帶 | y 範圍 | 用途 |
|---|---|---|
| 頂部標題/HUD | 0–390 | Logo、關卡/倍率/收益/餘額 |
| 戰鬥區 | 400–1240 | 背景、主角、怪物（**地面線 ≈ y1240**） |
| 操作 UI | 1300–1920 | 下注/決策/結算面板 |
> 角色主體**不得低於 y1240**（會被操作面板蓋到）。背景重要內容放中央,上下各約 12% 可能被遮。

| monster_001_idle … 005_idle | **768×768**（或 512×512）PNG32 | 主體約畫布 85%，水平置中 | **底部中央＝腳/底** | 約 420–480px | 右側 x≈780、底在地面線 y≈1240 | yes | **左** |

要點:
- **同一角色的多動作(idle/attack/hurt/defeat)畫布與 pivot 要一致**,這樣切換動作不會跳位。
- **逐格動畫做 sprite sheet 時**：依 `ART_CONTRACT` §七(v1.3)——**單邊 ≤ 4096px**;格數多必須用 **grid**(JSON 標 `columns`/`rows`),勿做超寬單列(會超過 H5 材質上限)。例:25 格 768px → 5×5 = 3840²。
- 主體底部貼齊畫布底部(留約 5–10% 透明邊即可),程式以「底部中央」對齊地面線。
- 螢幕目標高是程式縮放後的呈現大小;你只要交大圖(畫布內主體夠大、夠清晰),程式會 cover/scale。

## 5. 製程清單（每個素材照這跑）
1. **底圖**:Magnific / 手繪 / 其他生圖(參考 `prompts/magnific_prompts.md`;帶面向、側視、單一主體置中)。
2. **風格統一**:同一批用同一組風格描述,確保整體一致(`ART_DIRECTION_NOTES.md`)。
3. **去背**:角色/怪物/logo 去成透明(remove.bg / Photopea / rembg);背景不去背。
4. **裁切到交付畫布**:角色 768×768、主體置中、**腳貼底**;背景 1080×1920 滿版。
5. **命名**:= manifest 的 `file_name`(全小寫底線,如 `hero_idle.png`)。
6. **自我檢查**(對照下方清單)。
7. **放入** `Assets/final/`。
8. **工程端**:Godot 重新匯入(編輯器自動)/ H5 需重新 export 才生效。

## 6. 交付前自我檢查（每張打勾）
- [ ] 檔名 = manifest `file_name`（全小寫、底線、英數）
- [ ] 格式 PNG32（背景可 PNG24）；角色/怪物/logo 透明背景乾淨無白邊
- [ ] 面向正確（主角右、怪物左）
- [ ] 交付畫布尺寸正確（角色 768²、背景 1080×1920）
- [ ] 主體置中、腳貼畫布底（角色/怪物）
- [ ] 同角色多動作畫布/ pivot 一致
- [ ] 已放入 `Assets/final/`
- [ ] 在 manifest `notes` 標明像素風 or 非像素風（供 Codex 設 filter）
