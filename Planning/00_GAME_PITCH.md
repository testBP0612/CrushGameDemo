# 00 — 遊戲與比賽定位（Game Pitch）

## 比賽定位：這是「AI 工作流」展示，不只是遊戲
本作品參加公司內部「AI 遊戲創作大賽」。我們要展示的不是一款小遊戲本身，而是一套
**可複製、可多人協作的 AI 遊戲開發生產線**，以及避免 AI 失控的**治理機制**。

長期願景（老闆期待）：未來只要有企劃書，就能透過 AI 工作流產出可玩的遊戲原型。
本作品即此願景的**第一個可複製範本**。

## 展示三件事
1. 一款可玩的 H5 Godot 遊戲。
2. 一套檔案系統工作流：`企劃書 → AI 規劃 → Codex 任務卡 → Godot 實作 → 驗收迭代`。
3. 一套治理方式：防 context rot、防 AI 發散、防 AI 擅自改方向。

## 遊戲類型
手機直式 9:16 的**「風險撤離型自動戰鬥 H5 遊戲」**。
吸收 Crash Game / Chicken Dash 的「風險撤離」機制，但畫面是**可愛 2D 橫向 RPG 自動戰鬥**，
非博弈網站、非過馬路遊戲。下注以**虛擬金幣**呈現，淡化博弈感。

## 一局怎麼玩
用虛擬金幣下注 → 自動戰鬥擊敗怪物 → 倍率與收益提升 → 選「撤退領取」或「挑戰下一隻」
→ 失敗則本局損失。見好就收，還是再賭一把？

## MVP 範圍（必做）
- 9:16 H5 可玩閉環（下注 / 倍率 / 收益 / 撤退 / 戰敗 / 通關結算）。
- 自動戰鬥演出（連擊、受擊、死亡、轉場）+ placeholder。
- `LocalScoreService` + mock profile（餘額/最佳紀錄存 local）。
- H5 export 可在瀏覽器試玩。
- 前 5 隻怪物的正式美術接入路徑。

## 明確不做
大型會員系統、完整經濟系統、商城、裝備養成、抽卡、大型後端、多人同步、複雜反作弊、
真動作戰鬥、複雜怪物 AI、關卡編輯器、底部 Tab Bar、常駐資源列、設定/成就頁。

## Future（只預留架構，不在第一階段）
Google Login、排行榜、`ApiScoreService` / `FirebaseScoreService`、反作弊、6~10 隻怪物美術、更多特效。
