# 01 — 遊戲設計簡述（Game Design Brief）

> Codex 必讀 #1。本檔說明「要做什麼遊戲」。系統細節見 `02_SYSTEM_SPEC.md`，流程見 `03_STATE_MACHINE.md`。

## 一句話定義
手機直式 9:16 的**「風險撤離型自動戰鬥 H5 遊戲」**：玩家用虛擬金幣下注，挑戰一連串怪物，
每擊敗一隻倍率與收益提升，可隨時「撤退領取」或「挑戰下一隻」；挑戰失敗則本局損失。

## 核心循環（Core Loop）
```
下注 → 自動戰鬥演出 → 勝利 → 倍率/收益提升 → 玩家決策（撤退 or 繼續）
                         ↘ 失敗 → 本局結算 → 重新下注
```

## 樂趣點
- **風險張力**：愈往後倍率愈高、成功率愈低，「再賭一把 vs 落袋為安」的拉扯。
- **演出爽感**：可愛 2D 橫向 RPG 連擊、傷害數字、怪物死亡，包裝風險決策。
- **快節奏**：一局數十秒，適合 H5 內部試玩。

## 這「不是」什麼（避免 Codex 發散）
- 不是完整 RPG，沒有等級 / 技能樹 / 裝備養成。
- 不是真動作戰鬥，戰鬥是**自動演出**，玩家只做「下注 / 撤退 / 繼續」決策。
- 不是博弈網站包裝，視覺是可愛戰鬥；下注以**虛擬金幣**呈現，淡化博弈感。
- 不是過馬路 / Chicken Dash 的畫面，只借用「風險撤離」機制。

## 虛擬金幣經濟（最小集）
- 單一貨幣 `coins`，初始 `1000`（見 `Data/game_balance.json`）。
- 下注 → 戰鬥 → 撤退領取 or 失敗損失，餘額存 local（`LocalScoreService`）。
- **不做**商城、儲值、每日獎勵、抽卡。餘額歸零時可由「再來一局」重置（MVP 行為見 OPEN_QUESTIONS）。

## 數值來源
所有數值（倍率、成功率、下注上下限、怪物）皆讀 `Data/*.json`，**禁止寫死**。
詳見 `06_DATA_SCHEMA.md`。
